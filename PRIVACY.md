# Privacy Policy

Last Updated: February 2026

## 1. Introduction

This Privacy Policy explains how D365-MCP processes and handles data when interacting with Microsoft Dynamics 365 Finance and Operations and Azure services.

We are committed to maintaining enterprise-grade data protection and security standards.

---

## 2. Data Accessed

D365-MCP may access:

- Financial records
- Vendor and customer information
- Inventory data
- Transactional ERP data
- Operational business information

Access is strictly controlled by role-based permissions.

---

## 3. Data Collection

The agent does NOT:

- Independently collect personal data
- Store user passwords
- Share data with unauthorized third parties

Data is accessed only through secure Microsoft APIs.

---

## 4. Authentication and Security

D365-MCP uses:

- Microsoft Entra ID authentication
- OAuth 2.0 secure token exchange
- Azure role-based access control
- Encrypted HTTPS communication

All requests are validated before execution.

---

## 5. Data Storage

By default:

- The agent does not permanently store ERP data
- Data is processed in-memory for response generation
- Logs may be retained for audit and debugging purposes (depending on system configuration)

---

## 6. Data Sharing

We do not sell, rent, or distribute data.

Data remains within:

- Microsoft Azure infrastructure
- Microsoft Dynamics 365 environment
- Authorized enterprise systems

---

## 7. Compliance

The agent is designed to align with:

- Enterprise IT governance policies
- Data protection best practices
- Applicable regulatory requirements (e.g., GDPR principles where applicable)

---

## 8. User Responsibilities

Users must:

- Ensure proper role assignments
- Protect authentication credentials
- Follow internal security guidelines

---

## 9. Updates to This Policy

This Privacy Policy may be updated periodically.

Continued use of the agent indicates acceptance of the updated policy.
